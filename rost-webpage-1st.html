🎯 Project Background

분리수거, 왜 제대로 안 될까요?

  • 복잡한 분류 기준 — 재질별, 상태별 판단 필요
  • 수작업 의존 — 인식, 분류, 배출 전 과정이 수동
  • 낮은 참여율 — 번거로움으로 인한 분리수거 기피

💡 Our Solution

End-to-End Automated Sorting

사용자는 쓰레기를 놓기만 하면,
Detection → Classification → Disposal
전 과정이 자동으로 수행됩니다.

Perception: Challenges

Estimation과 Control의 품질을 결정하는 관문

❌ Technical Challenges

  • RGB 카메라 → 거리 정보 없음 + 조명 변화에 취약
  • 픽셀 좌표 → 로봇은 3D 월드 좌표가 필요
  • 불규칙한 형태 → 집기 위한 각도 정보 필요

✓ Our Approach

  • RealSense Depth 카메라 — 조명 영향 최소화
  • 좌표 변환 — 픽셀 → 카메라 → 월드 좌표
  • 회전된 BBox — 그리퍼 각도 정보 제공
💡 핵심: Perception에서 판단을 뒤로 미루지 않기

🔬 Core Technology: DBSCAN Clustering

Density-Based Spatial Clustering of Applications with Noise

💡 Why DBSCAN?

  • 클러스터 개수를 미리 지정할 필요 없음
  • 불규칙한 형태도 밀도 기반으로 하나로 묶음
  • 의미 없는 픽셀/잡음은 노이즈로 자동 제거

📊 Key Insight

  • "개수를 맞히는 문제"가 아니라
  • "의미 있는 덩어리를 찾는 문제"로 전환
  • → 각 클러스터 = 하나의 물체

Perception: Solutions

DBSCAN 기반 탐지, 필터링, 좌표 변환

🎨 Dual BBox Strategy

Green BBox

  • Method: Depth → World 좌표 → DBSCAN 클러스터링
  • Target: 깊이 인식이 정상적인 물체
  • Feature: 빠른 탐지, 넓은 커버리지
✓ 깊이 기반 실체를 우선 탐지

Blue BBox

  • Method: RGB + Depth hole + Gradient 분석
  • Target: 투명/반사 물체 (플라스틱 등)
  • Feature: 컨투어 기반 박스 생성
✓ 깊이 취약 영역을 보완
💡 파란 박스 중심이 초록 박스 내부에 포함되면 → 중복 제거

Estimation: AI Classification

Gemini API를 활용한 멀티모달 쓰레기 분류

❌ Initial Approach: YOLO

  • 처음엔 YOLO 모델로 쓰레기 학습 시도
  • 새로운 쓰레기 종류 추가 시 재학습 필수
  • 학습 데이터 수집 & 라벨링에 많은 시간 소요

✓ Our Choice: Gemini API

  • Zero-shot Classification — 별도 학습 없이 즉시 분류
  • 새로운 카테고리 추가 = 프롬프트 수정만으로 OK
  • 멀티모달 (이미지 + 텍스트) 이해 능력 활용
  • 빠른 프로토타이핑 & 확장성 확보

🤖 Gemini API

Google의 멀티모달 AI 모델을 활용하여 이미지 기반 쓰레기 종류를 자동 판별합니다.
